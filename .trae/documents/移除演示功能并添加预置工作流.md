# 功能调整计划：移除旧演示并添加预置工作流

我将执行以下操作来满足您的需求：

## 1. 移除旧的演示功能 (前端)
*   **目标**: 移除“点击显示演示会弹出智能客服助手”的功能。
*   **操作**:
    *   从 `frontend/src/views/WorkflowView.vue` 中移除 `WorkflowDemo` 组件的引用及相关触发逻辑（如按钮或自动弹窗）。
    *   删除 `frontend/src/components/workflow/WorkflowDemo.vue` 组件文件（如果不再被其他地方引用）。

## 2. 添加预置示例工作流 (后端)
*   **目标**: 系统启动时，自动创建一个“示例工作流 - AI智能助手”，用户打开列表即可见。
*   **实现方案**:
    *   **创建 Seed Service**: 在后端创建一个 `WorkflowSeedService`，用于在应用启动时初始化数据。
    *   **定义示例数据**: 将“AI智能助手”的完整图数据（节点、连线、配置）定义为常量。
    *   **启动检查**: 在 `onModuleInit` 生命周期钩子中，检查数据库是否为空（或是否存在特定名称的示例工作流）。如果不存在，则自动创建。
*   **技术细节**:
    *   使用 NestJS 的 `OnModuleInit` 接口。
    *   利用 `WorkflowService` 复用创建逻辑。
    *   示例工作流将包含：Start 节点 -> LLM 节点 (配置好 Qwen Prompt) -> End 节点。

## 3. 验证
*   重启前后端服务。
*   访问前端，确认旧的演示弹窗不再出现。
*   查看工作流列表，确认是否自动出现了一个名为“示例工作流 - AI智能助手”的新条目。
*   点击进入并运行，验证其是否可用。
