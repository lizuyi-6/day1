## 问题分析

页面中的终端没有显示执行过程的日志，问题在于 WebSocket 连接的命名空间配置。

**后端配置** (`workflow.gateway.ts` 第 11-16 行)：
```typescript
@WebSocketGateway({
  cors: {
    origin: '*',
  },
  namespace: 'workflow',  // 后端使用 /workflow 命名空间
})
```

**前端配置** (`WorkflowView.vue` 第 1504 行)：
```typescript
socket.value = io('http://localhost:3001', {  // ❌ 没有指定命名空间
  transports: ['websocket', 'polling'],
})
```

## 修复方案

修改前端 Socket.IO 连接，添加正确的命名空间：

```typescript
// 在 WorkflowView.vue 第 1504 行
socket.value = io('http://localhost:3001/workflow', {
  transports: ['websocket', 'polling'],
})
```

这样前端就能正确接收来自后端的 WebSocket 消息，包括终端日志输出。
