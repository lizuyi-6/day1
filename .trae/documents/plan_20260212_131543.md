## 问题分析

前端发送的数据包含 `id` 字段：

```typescript
{
  id: record.id,  // ❌ 后端 DTO 没有这个字段
  workflowId: record.workflowId,
  status: record.status,
  ...
}
```

但后端 `CreateExecutionHistoryDto` 没有定义 `id` 字段，且验证配置 `forbidNonWhitelisted: true` 会拒绝任何不在白名单中的字段。

## 修复方案

修改 `frontend/src/views/WorkflowView.vue` 第 777-786 行，移除 `id` 字段：

```typescript
// 修改前
const backendRecord: BackendExecutionHistory = {
  id: record.id,  // 删除这行
  workflowId: record.workflowId,
  ...
}

// 修改后
const backendRecord: CreateExecutionHistoryDto = {
  workflowId: record.workflowId,
  ...
}
```

这样就能正常保存执行历史了。
