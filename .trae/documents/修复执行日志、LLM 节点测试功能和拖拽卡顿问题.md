## 问题分析

### 问题1: 执行日志不显示内容
- `WorkflowView.vue` 中的日志处理逻辑可能无法正确解析后端返回的数据
- 日志没有包含输入数据

### 问题2: LLM 节点缺少测试功能
- 后端有 `debugNode` API，但前端检查器没有测试按钮

### 问题3: 输出数据格式问题
- 后端返回的数据格式可能与前端期望的不一致

### 问题4: 拖拽卡顿
- Vue Flow 响应式更新可能导致性能问题

---

## 修复计划

### 1. 修复执行日志显示
- 在创建日志时添加输入数据
- 确保正确解析后端返回的 `nodeOutputs`

### 2. 添加 LLM 节点测试功能
- 在 `WorkflowInspector.vue` 中添加"测试节点"按钮
- 调用后端 `debugNode` API
- 显示测试结果

### 3. 修复输出数据解析
- 确保 `nodeOutputs` 中的键值对正确映射到日志

### 4. 优化拖拽性能
- 检查节点数据的响应式更新
- 使用 `shallowRef` 优化
