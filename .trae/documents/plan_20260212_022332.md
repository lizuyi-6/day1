# 修复工作流编辑器页面卡顿

## 问题根因
1. 路由使用 `transition: 'zoom-in'`，但 App.vue 中缺少对应的 CSS 动画定义
2. WorkflowView.vue 初始化与页面转换动画重叠（16个异步组件 + 数据加载 + fitView）

## 优化方案（不改变现有动画风格）

### 修改 1：App.vue - 添加 `zoom-in` 动画样式
- 在现有 7 种动画后添加 `zoom-in` 动画
- 使用与其他动画相同的缓动函数 (`var(--ease-out-expo)`)
- 保持相似的视觉效果（缩放 + 模糊 + 平移）

### 修改 2：WorkflowView.vue - 优化初始化时机
- 检测页面转换状态（通过 `isTransitioning` 状态或监听路由）
- 将 `onPaneReady` 中的工作流加载延迟到页面转换完成后
- 使用 `setTimeout` 或 `nextTick` 确保动画完成后再加载

## 预期效果
- 页面切换流畅，无卡顿
- 保持所有现有动画风格不变
- 工作流编辑器正常加载
