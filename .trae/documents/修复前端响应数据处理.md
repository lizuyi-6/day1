## 问题分析

后端返回的数据结构：
```typescript
{
  items: [...],  // 工作流数组在这里
  total: 1,
  page: 1,
  limit: 20,
  totalPages: 1
}
```

但前端代码在查找 `workflows` 或 `data` 属性：
```typescript
if (Array.isArray(response.data.workflows)) {
  workflowsArray = response.data.workflows
} else if (Array.isArray(response.data.data)) {
  workflowsArray = response.data.data
}
```

**结果**：`workflowsArray` 最终是空数组，导致前端显示"没有工作流"，但数据库中实际存在数据！

## 修复方案

修改 `frontend/src/services/workflowService.ts` 中的 `getAllWorkflows` 函数，添加对 `items` 属性的处理：

```typescript
// 第 131-138 行，修改为：
if (Array.isArray(response.data.workflows)) {
  workflowsArray = response.data.workflows
} else if (Array.isArray(response.data.data)) {
  workflowsArray = response.data.data
} else if (Array.isArray(response.data.items)) {
  // 处理后端的 items 格式
  workflowsArray = response.data.items
}
```

这样就能正确显示工作流列表了。修复后，用户应该能看到之前"消失"的工作流。
