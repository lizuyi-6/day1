#### 问题分析
用户反馈示例工作流仍未显示，可能原因：
1. `@pane-ready` 事件在 Vue Flow 1.48.2 版本可能不支持或已更改
2. Vue Flow 组件初始化时机问题
3. `addNodes` 和 `addEdges` 可能在 Vue Flow 未完全初始化时调用

### 修复方案

#### 方案 1：使用 `onInit` 事件（推荐）
检查 Vue Flow 文档，将 `@pane-ready` 改为 `@init` 事件，这是更可靠的生命周期事件。

#### 方案 2：在 `onMounted` 中直接初始化
移除 `onPaneReady` 事件监听，直接在 Vue 组件的 `onMounted` 中检查路由参数并初始化示例工作流。

#### 方案 3：使用 `watch` 监听路由
监听 `route.params.id` 的变化，当 ID 不存在时初始化示例工作流。

### 实施步骤

1. **添加调试日志**
   - 在 `onPaneReady` 中添加 console.log
   - 在 `onMounted` 中添加 console.log

2. **检查 Vue Flow 事件**
   - 查看 Vue Flow 1.48.2 的文档确认事件名称
   - 尝试使用 `@init` 替代 `@pane-ready`

3. **备用方案**
   - 如果事件方式不工作，在 `onMounted` 中直接初始化
   - 或使用 `watch` 监听路由变化

4. **验证修复**
   - 测试创建新工作流是否显示示例
   - 检查浏览器控制台是否有错误
